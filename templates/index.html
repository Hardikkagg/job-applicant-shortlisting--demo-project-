Algorithm Reference

How TalentLens processes your data under the hood

Quick Sort

O(n log n) average · O(n²) worst

Used to sort all candidates by score, experience, or salary. Pivot is chosen from the middle to reduce worst-case on already-sorted data. Each call partitions the list into left ≥ pivot and right < pivot, then recursively sorts each partition.

quickSort(arr, key):
  if len(arr) <= 1: return arr
  pivot = arr[len//2]
  left  = [x for x in arr if x[key] >= pivot[key]]
  right = [x for x in arr if x[key] <  pivot[key]]
  return quickSort(left) + [pivot] + quickSort(right)

Binary Search

O(log n) to find · O(k) for duplicates

Requires pre-sorted input. Narrows the search space by half each iteration. Once a match is found, expands left/right to collect all candidates sharing the same value (handles ties elegantly).

binarySearch(arr, target, key):
  lo, hi = 0, len(arr)-1
  while lo <= hi:
    mid = (lo+hi) // 2
    if arr[mid][key] == target: return mid
    elif arr[mid][key] > target: lo = mid+1
    else: hi = mid-1

Recursive Algorithms

Three recursive functions in use

  • recursive_skill_filter — tail-recursively walks the candidate list, checking each one for all required skills via a nested recursion (_all_skills_present).
  • recursive_rank_build — assigns rank numbers while handling ties (equal score → same rank).
  • _all_skills_present — inner helper: checks required skills one by one recursively.

Auto-Score Formula

Weighted multi-factor scoring

Automatically computes a 0–100 score for every candidate:

score = (experience × 2)
      + (skills_count × 3)
      − (salary / 10000)
      clamped to [0, 100]

Higher experience and more skills push the score up; higher salary expectation has a mild negative weight.
